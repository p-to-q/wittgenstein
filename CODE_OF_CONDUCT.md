# Code of Conduct

Wittgenstein follows version 2.1 of the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
This page summarises the parts most relevant to an early-stage, research-leaning
open-source project.

## The short version

- **Be direct, stay kind.** Technical disagreement is welcome; personal attacks are not.
- **Assume good faith.** Most issues come from missing context, not bad intent. Ask before
  escalating.
- **Credit work honestly.** Attribute ideas, benchmarks, and code to their sources. When
  porting an idea from a paper or another project, link it.
- **Keep the bar research-honest.** If a result is a dry-run proxy, say so. If a feature is
  a stub, label it `⚠️ Experimental` in docs. Don't oversell.
- **No harassment, in any channel.** Issues, PRs, discussions, Discord, email, in-person —
  the rules are the same.

## Scope

Applies to every space the project owns: the GitHub repo, discussions, linked chat
channels, demo venues, and any one-on-one communication about the project.

## Reporting

1. Open a private security advisory on the repo ("Report a vulnerability" flow) **if the
   incident also involves a security concern.**
2. Otherwise, email the maintainer listed in `CODEOWNERS` (`@max`) via the address in your
   GitHub profile, or DM on the project's primary chat channel.
3. We aim to acknowledge within 5 business days.

Retaliation against a reporter is itself a violation.

## Enforcement

First incidents get a private, specific note. Repeated or severe incidents result in a
temporary or permanent ban from the project's spaces. Decisions are made by the project
maintainers and are final for the duration of the action; they can be appealed to the
core maintainer group in writing.
